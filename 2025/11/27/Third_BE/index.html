【DYA3】C++聊天室：Redis缓存层实现与架构优化

📅 今日里程碑：引入 Redis 高速缓存

为了解决 MySQL 在高并发下的读写瓶颈，以及提升用户登录时的历史消息加载速度，今天我正式引入了 Redis 作为缓存层。

现在的系统架构已经升级为： 客户端 (Qt/Python) <–> C++ 服务端 <–> Redis (热数据) + MySQL (持久化)

🛠️ 核心技术实现

1. 双写策略 (Write-Through 变种)

每当服务端收到一条新消息时，会同时进行两个操作：

  • 写入 MySQL：保证数据绝对不丢失（持久化）。

  • 写入 Redis List：使用 LPUSH 命令塞入 chat_history 列表头部，并使用 LTRIM 保持列表长度为 20，确保内存中只存最新的热点数据。

2. 读写分离与降级

用户登录获取历史记录时：

  • 优先读 Redis：使用 LRANGE 全量取出内存中的 20 条记录，速度极快（微秒级）。加入能读写图片的能力。

  • 降级方案：如果 Redis 挂了或者没数据，自动降级去查 MySQL，保证服务可用性。

3. 大文件策略优化

在处理图片消息时，为了防止 Base64 编码的大图阻塞网络或撑爆客户端内存，我在服务端增加了智能判断：

  • 实时转发：大图依然可以实时发送，保证聊天体验。

  • 历史回放：超过 2MB 的图片在历史记录中会被替换为 [历史图片] 提示文本，极大提升了登录速度。

💻 当前代码状态 (Server.cpp)

目前服务端 (server.cpp) 已经集成了以下库：

  • <mysql/mysql.h>: MySQL C API

  • <hiredis/hiredis.h>: Redis C API

  • <nlohmann/json.hpp>: JSON 序列化

  • <thread>, <mutex>: 多线程并发
