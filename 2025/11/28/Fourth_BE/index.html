【实战Day4】C++聊天室进化：Qt GUI 重构、群聊私聊分离与 AI 深度集成

📅 今日概览

如果说前两天是在打地基（Socket, Epoll, MySQL, Redis），那么今天就是“装修入住”的高光时刻。
我彻底重构了客户端，抛弃了简陋的命令行，用 Qt 写出了一个支持 群聊/私聊分组、图片发送、好友管理 的现代化 IM 界面。同时，我还通过 Python 微服务的方式，无缝接入了 DeepSeek AI，让聊天室拥有了一个 24 小时在线的智能陪聊。

🖥️ 1. Qt 客户端重构：像 QQ 一样聊天

为了实现“私聊”和“群聊”的分离，我对 UI 进行了修改。

核心架构升级

  • 列表控件升级：从简单的 QListWidget 升级为 QTreeWidget，实现了“我的群组”和“私聊好友”的折叠分组显示。
  • 数据驱动 UI：利用 QTreeWidgetItem::setData(Qt::UserRole, ...) 存储元数据（如 Socket ID、聊天类型），实现了点击列表项自动切换聊天上下文。

关键功能实现

  1. UI 布局：采用经典的“左侧列表 + 右侧聊天框 + 底部发送栏”的三段式布局。
  2. 群聊/私聊分离：
    • 发送消息时，根据当前选中的列表项，自动封装 group_msg 或 private_msg 类型的 JSON 包。
    • 接收消息时，根据协议类型 (group_chat / private_chat) 自动将消息路由到对应的显示逻辑，并支持 HTML 富文本渲染。
  3. 好友系统：
    • 实现了 “添加好友” 功能，输入用户名即可发起请求。
    • 实现了 “自动加好友”：当收到陌生人的私聊消息时，自动将其添加到左侧好友列表中。
  4. 图片支持：完美继承了之前的 Base64 图片发送逻辑，并限制了显示宽度，防止大图刷屏。

踩坑记录：

  • 编译报错 QTcpSocket file not found：需要在 .pro 文件中添加 QT += network 并重新运行 qmake。
  • 图标资源报错：Qt 6 中没有 SP_UserIcon，改用 SP_ComputerIcon 暂时替代。

🧠 2. AI 接入：Python 微服务架构

为了让聊天室更有趣，我没有在 C++ 服务端里死磕 HTTP 库，而是采用 微服务 的思路，写了一个独立的 Python 脚本作为 AI 机器人。

架构设计

  • 独立进程：ai_bot.py 作为一个普通的 TCP 客户端连接 C++ 服务器。
  • 监听与触发：机器人监听所有群消息，当检测到 @AI 关键词时触发逻辑。
  • AI 大脑：调用 DeepSeek API 获取回复，并将结果封装成 JSON 发回聊天室。

代码思考

这种设计使得 AI 在思考（网络请求）时，机器人依然能接收并处理其他人的消息，实现了高并发处理。

⚙️ 3. 服务端逻辑完善 (Epoll + 业务逻辑)

服务端 (server_epoll.cpp) 是整个系统的调度中心，今天主要完善了业务分发逻辑。

  • 协议扩展：新增了 create_group (建群), join_group (加群), private_msg (私聊) 等指令的处理。

  • 状态管理：维护了全局的 groups (群组表) 和 online_users (在线用户表)，并加了互斥锁 mutex 保证线程安全。

  • 回执优化：在建群成功后，将群 ID 和群名称打包返回给客户端，方便前端自动刷新列表。

🐛 4. 环境与调试踩坑

今天的开发中也遇到了一些“经典的”环境问题：

  1. WSL 连接拒绝 (WinError 10061)：

    • 原因：WSL 重启后 MySQL 服务未自动启动。

    • 解决：sudo service mysql start。

  2. Python 依赖问题 (externally-managed-environment)：

    • 原因：Ubuntu 24.04 对 pip 安装系统包有限制。

    • 解决：优先使用 sudo apt install python3-requests，或者使用虚拟环境。

  3. 幽灵进程：

    • 现象：服务端报错端口被占用，Qt 连上就断。

    • 解决：sudo fuser -k 8888/tcp 强杀旧进程，重新编译运行。

📝 总结

现在的聊天室已经具备了 工业级 IM 软件 的雏形：

  • 后端：Epoll 高并发 + Redis 缓存 + MySQL 持久化。

  • 前端：Qt 图形界面 + 多媒体支持 + 好友/群组管理。

  • 智能：DeepSeek AI 实时陪聊。

  • 心得：今天写下来感觉还是很有收获的，感觉必须区深入了解原理了，计组计网这些理论知识真的太重要了，我觉得不止要学会业务上的东西，我觉得我现在写的就应该当个玩具，但是要坚持学习，哪怕不往算法||深度学习方向走，也应该有科班出来的底子，而不是什么都不会，只会利用工具，要会根据需求进行分析，解决实际问题。

Next Step: 接下来的计划是进一步优化 UI 美观度（使用 QSS），并尝试将服务端部署到真正的云服务器上，邀请朋友公网实测。
