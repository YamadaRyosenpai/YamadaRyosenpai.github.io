从零搭建：我的第一个 Linux C++ 公网聊天室 （这是一个测试，作为一个学习记录）

前言

作为一个计算机专业的学生，我一直想搞懂后端到底是怎么工作的。最近我利用 WSL2 (Ubuntu) 和 C++，从零手写了一个支持多线程、数据库存储的聊天室，并通过 Pinggy 实现了公网访问。

这是我的架构图：

1. 环境搭建：WSL 与 踩坑

在 Windows 下开发 Linux 程序，WSL2 是最好的选择。
但我遇到了一个经典报错 Error: 0xd000000d。
而且无法通过开启Windows功能中的Hyper-V来解决问题，依旧报错

解决方案：
需要通过命令行强制开启 Hypervisor：
```cmd
bcdedit /set hypervisorlaunchtype auto
```

2. 核心代码：多线程与 Socket

为了让服务器能同时处理多个人的消息，我使用了 std::thread。

核心逻辑如下：
```cpp
// 简单的广播逻辑
void broadcast_message(string message, int sender_fd) {
lock_guard lock(clients_mtx); // 加锁防止冲突
for (int client_fd : all_clients) {
send(client_fd, message.c_str(), message.length(), 0);
}
}
```

3. 数据持久化：接入 MySQL

聊天记录不能发完就丢。我设计了一张 messages 表来存储记录。
遇到的难点是 C++ 连接 MySQL 需要安装 libmysqlclient-dev，并且编译时要加上链接库参数。

4. 公网穿透：让手机连上电脑

因为没有公网 IP，我使用了 Pinggy。
只需要一条命令就能把本地 8888 端口暴露给公网：
```bash
ssh -p 443 -R0:localhost:8888 tcp@a.pinggy.io
```

总结

通过这个项目，我理解了 TCP 粘包、多线程竞争以及数据库连接的基本原理。下一步我计划实现数据库连接池，进一步提升性能。
